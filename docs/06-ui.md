## Routes

React Router v7:

- `/` — redirects to the most recently updated profile, or `/profiles/new` if none exist.
- `/profiles/:id` — profile view (sidebar + detail panes). Default landing for a known profile.
- `/profiles/:id/sessions/:sessionId` — same layout, but stats panel displays the historical session read-only.
- `/profiles/new` — blank create form (editable before save).
- `/settings` — global settings screen (binary path, models dir, port, retention limit).

## Component responsibilities

### TopBar

- Shows global server status (idle / starting / running / stopping / stopped / crashed) with a colored dot + text.
- If running, shows the active profile's name next to the dot.
- Start button: disabled unless a profile is selected AND state is `idle | stopped | crashed`.
- Stop button: enabled only when state is `starting | running`.
- Settings cog navigates to `/settings`.

### ProfileSidebar

- Top section: user profiles, sorted by `updatedAt` desc.
- Currently selected profile has a visible active state.
- Currently running profile has a small running-indicator dot (even if not selected).
- "+ New" button → navigates to `/profiles/new`.
- Bottom section: predefined templates grouped under a "Templates" heading. Each item has a "Clone" icon-only button (per `CLAUDE.md` §8: icon-only row actions with `title`/`aria-label`). Clicking clone creates a user profile and navigates to it.

### ProfileConfigCard / ProfileConfigForm

- Card displays: name, description, model file, args (rendered as chips), timestamps.
- "Edit" icon-only button toggles into form mode.
- Form:
  - Name (required, 1–120 chars).
  - Description (optional multi-line).
  - Model file: dropdown sourced from reading `modelsDir` server-side (new endpoint `GET /api/models` — mentioned here as a UI need; add to `docs/04-api.md` when building Phase 3). Falls back to text input if listing fails.
  - Args: editable list of strings (add / remove rows), with client-side validation matching server: no `--model | -m | --host | --port`.
  - Save / Cancel. Save calls `PUT /api/profiles/:id`.
- Saving while the profile is the active running profile is allowed but prompts a confirm dialog: "Restart required for changes to take effect." No auto-restart in v1.

### StatsPanel

- Container: decides which session to render.
  - Default: the current live session if the active profile is running, otherwise the most recent past session, otherwise an empty state.
  - `SessionPicker` lets the user pick any session (including "Live"). When a historical session is picked, the URL updates to `/profiles/:id/sessions/:sessionId`.
- Passes the session's metrics (from `useMetricsStore` for live, from `fetch` for historical) to child cards.
- Cards laid out in a 12-col grid:
  - Row 1: `TotalsCard` (col-span 5) · `MemoryBudgetCard` (col-span 7).
  - Row 2: `ThroughputChart` (col-span 12).
  - Row 3: `RecentRequestsTable` (col-span 7) · `PromptCacheCard` (col-span 5).
  - Row 4: `StartupInfoCard` (col-span 8) · `ErrorsPanel` (col-span 4). Collapses if no errors.

### LogPanel

- Virtualized scrolling list (windowed render; library-free implementation using row height estimation).
- Auto-scroll sticks to the bottom if the user is already at the bottom; breaks when the user scrolls up and re-sticks when they jump back.
- Noise toggle: default ON; hides lines with `noise: true`.
- Stream toggle: Live ⟷ Historical (driven by the same `SessionPicker` decision as the stats panel).
- Search / filter input (client-side substring filter over visible lines).
- Each log line is colored by stream: stdout default, stderr slightly darker/red.

## Empty / loading / error states

| Screen | Empty | Loading | Error |
|---|---|---|---|
| App on first launch (no settings) | Redirect to `/settings` with a banner: "Set your llama-server binary and models directory to get started." | Skeleton in TopBar. | Banner with retry. |
| Sidebar with no user profiles | Shows "No profiles yet. Pick a template below or create one from scratch." | Skeleton list items. | — |
| ProfileDetail before a profile is selected | Placeholder: "Select a profile to view its configuration and sessions." | — | — |
| ProfileDetail when selected profile has zero sessions | Stats panel shows "Run this profile to see statistics." Logs show "No logs yet." | — | Error boundary content with retry. |
| StatsPanel in historical mode | If `metrics.json` is missing/incomplete: "This session has no parsed metrics (parser did not recognize any events)." | Skeleton cards. | Error boundary. |
| LogPanel loading a large historical log | Streaming spinner in the header while paging. | — | Error banner; partial lines rendered. |

## Keyboard shortcuts (nice-to-have, Phase 11)

- `⌘K` / `Ctrl+K` — focus profile sidebar search.
- `⌘↵` / `Ctrl+Enter` — save form.
- `Esc` — cancel form / close dialog.
- `.` — toggle log noise filter.
- `/` — focus log search input.

## Accessibility

- All row actions are icon buttons with `aria-label` and `title` (`CLAUDE.md` §8).
- Color is never the only signal of status (the status indicator also carries text).
- Contrast passes WCAG AA.
- Charts expose an accessible table alternative via a "View data" toggle.
- Focus order follows the visual layout; all interactive elements reachable via keyboard.

## Theming

Out-of-scope for v1. Tailwind's default palette is used as-is. System `prefers-color-scheme` may be honored later; v1 ships light or dark (TBD in Phase 11) — not both.

## Design constraints reminder

From `CLAUDE.md` §8:

- Row actions are icon-only with `title` + `aria-label`.
- Persist per row (each profile save/delete is its own mutation).
- Error boundary per major pane.
- Forms initialise from a typed model, not `{}`.
- Effect deps are stable primitives.
- Framework-provided functionality is used, not reimplemented (react-router for navigation, react's error boundaries for isolation, tailwind utilities for styling — no custom CSS files beyond the global entry).
